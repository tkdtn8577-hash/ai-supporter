# PRD: 1인 웹 에이전시 랜딩페이지 (코드네임: FOLIO)

> 홈페이지 제작 1인 에이전시의 포트폴리오 + 문의 수집 랜딩페이지. Apple 스타일 다크 테마, Vercel + Supabase 기반.

---

## 1. 제품 비전

- **한 줄 요약**: 방문자가 3초 안에 "이 사람한테 맡기고 싶다"는 인상을 받고, 바로 문의로 이어지는 랜딩페이지
- **왜 만드는가**: 1인 에이전시의 신뢰도는 첫인상에서 결정됨. 구구절절한 설명보다 압도적인 비주얼과 명확한 CTA로 전환율을 극대화. 문의는 이메일/카카오 DM이 아닌 폼으로 수집해 관리.
- **타겟 사용자**:
  - **방문자**: 홈페이지 제작을 원하는 소상공인, 스타트업, 개인 사업자
  - **관리자**: 에이전시 대표 (1인, 문의 확인 및 상태 관리)

---

## 2. 핵심 원칙

| 원칙 | 설명 |
|------|------|
| 첫인상 임팩트 | 히어로 섹션에서 즉각적인 신뢰와 감탄을 유발 |
| 마찰 없는 문의 | 폼 필드 최소화, 모바일에서도 30초 안에 제출 완료 |
| 다크 Apple 스타일 | 검정 배경, 샌프란시스코 계열 서체, 정제된 여백, 부드러운 인터랙션 |
| 자체 인프라 | Vercel + Supabase 무료 티어, 외부 폼 서비스 없음 |
| 단일 페이지 집중 | 복잡한 멀티페이지 없이 스크롤 한 번으로 모든 정보 전달 |

## 3. 기능 요구사항

### P0 — Must Have (MVP)

#### F1: 히어로 섹션
- **유저 스토리**: 방문자가 사이트 진입 즉시 "무엇을 하는 곳인지", "믿을 수 있는지" 판단할 수 있어야 한다.
- **상세 요구사항**:
  - 풀스크린 레이아웃 (100vh)
  - 대형 타이포그래피 헤드라인 (주요 서비스 한 줄 압축)
  - 서브텍스트: 업력 또는 차별점 한 줄
  - CTA 버튼 1개: `프로젝트 문의하기` → Contact 섹션 앵커 스크롤
  - 배경: 순흑(#000) 또는 다크 그라데이션
  - 스크롤 유도 인디케이터 (하단 화살표 or 텍스트)
- **기술 결정**: Framer Motion `fadeInUp` 진입 애니메이션

#### F2: 서비스 섹션
- **유저 스토리**: 방문자가 어떤 서비스를 받을 수 있는지 한눈에 파악할 수 있어야 한다.
- **상세 요구사항**:
  - 카드 3개: `웹사이트 제작` / `랜딩페이지` / `쇼핑몰`
  - 각 카드: 아이콘 + 제목 + 2줄 설명
  - hover 시 카드 테두리 글로우 효과 (Apple-style)
  - 모바일: 1열 세로 스택

#### F3: 포트폴리오 섹션
- **유저 스토리**: 방문자가 실제 결과물을 보고 제작 수준을 판단할 수 있어야 한다.
- **상세 요구사항**:
  - 작업물 카드 6개 (초기 기준, 코드 수정으로 추가 가능)
  - 각 카드: 썸네일 이미지 + 프로젝트명 + 카테고리 태그
  - 클릭 시 외부 링크(새 탭) 또는 이미지 라이트박스 모달
  - 데이터 관리: `works.ts` 파일에 하드코딩 (재배포로 업데이트)
  - 모바일: 1열, PC: 2열 그리드
- **기술 결정**: 포트폴리오 데이터를 DB가 아닌 코드로 관리 (업데이트 빈도 낮음, 구현 단순화)

#### F4: 프로세스 섹션
- **유저 스토리**: 방문자가 "계약 후 어떻게 진행되는지" 불안 없이 이해할 수 있어야 한다.
- **상세 요구사항**:
  - 5단계 수평 타임라인: `상담` → `기획/디자인` → `개발` → `납품` → `유지보수`
  - 각 단계: 번호 + 단계명 + 한 줄 설명
  - 스크롤 진입 시 좌→우 순차 등장 애니메이션
  - 모바일: 수직 타임라인으로 전환

#### F5: 가격 안내 섹션
- **유저 스토리**: 방문자가 예산 범위를 미리 확인하고 문의 여부를 결정할 수 있어야 한다.
- **상세 요구사항**:
  - 서비스별 시작가 표시: `랜딩페이지 150만원~`, `웹사이트 300만원~`, `쇼핑몰 500만원~`
  - 가격 아래 단서: "요구사항에 따라 변동, 문의 시 정확한 견적 제공"
  - 구체적인 패키지 티어 없음 (협상 유연성 유지)
  - CTA 버튼: `견적 문의하기` → Contact 섹션으로 스크롤

#### F6: 문의 폼 (Contact)
- **유저 스토리**: 방문자가 페이지를 떠나지 않고 바로 문의를 남길 수 있어야 한다.
- **상세 요구사항**:
  - 필드: `이름`(필수) / `연락처`(필수) / `이메일`(선택) / `예산`(선택, 드롭다운) / `문의 내용`(필수)
  - 예산 드롭다운 옵션: `150만원 미만` / `150~300만원` / `300~500만원` / `500만원 이상` / `미정`
  - 제출 시: Supabase `inquiries` 테이블에 저장
  - 성공 시: 폼 영역을 "문의가 접수되었습니다. 24시간 내 연락드리겠습니다." 메시지로 교체
  - 실패 시: 인라인 에러 메시지 표시 (페이지 이동 없음)
  - 유효성 검사: 필수 필드 공란, 연락처 형식 (클라이언트 사이드)

#### F7: 어드민 페이지 (`/admin`)
- **유저 스토리**: 에이전시 대표가 수신된 문의를 한 곳에서 확인하고 상태를 관리할 수 있어야 한다.
- **상세 요구사항**:
  - **접근 인증**: Supabase Magic Link (이메일 `tkdtn8577@gmail.com`으로 로그인 링크 발송)
  - 미인증 상태에서 `/admin` 접근 시 로그인 페이지로 리다이렉트
  - **문의 목록 테이블**: 접수일 / 이름 / 연락처 / 예산 / 상태 / 문의 내용 미리보기
  - **상태 관리**: `신규` → `확인` → `완료` (클릭으로 인라인 변경)
  - **정렬**: 최신순 기본, 상태별 필터
  - **문의 상세**: 행 클릭 시 전체 내용 모달 또는 확장 패널
  - 삭제 기능 (확인 다이얼로그 포함)
- **기술 결정**: Supabase Auth Magic Link 사용. 세션은 Supabase 클라이언트가 자동 관리.

## 4. 기술 스택

| 레이어 | 기술 | 이유 |
|--------|------|------|
| 프레임워크 | Next.js 14 (App Router) | Vercel 최적화, SSR/SSG, API 라우트 |
| 스타일 | Tailwind CSS + CSS variables | 다크 테마 관리, 반응형 빠른 구현 |
| 애니메이션 | Framer Motion | Apple-style 스크롤 인터랙션 |
| DB | Supabase (PostgreSQL) | 문의 저장, Auth, 무료 티어 |
| 인증 | Supabase Auth (Magic Link) | 어드민 전용, 구현 단순 |
| 배포 | Vercel Hobby | HTTPS 자동, 글로벌 CDN, 무료 |
| 포트폴리오 | `works.ts` 하드코딩 | 업데이트 빈도 낮음, DB 불필요 |

### 아키텍처 원칙
- 단일 페이지 (랜딩) + 어드민 페이지만 존재. 복잡한 라우팅 없음.
- 문의 폼은 Server Action 또는 API Route로 처리 (클라이언트에 Supabase key 미노출)
- 어드민은 Supabase RLS로 인증된 사용자만 데이터 접근 가능

### 데이터 모델

```sql
-- 고객 문의
CREATE TABLE inquiries (
  id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  name        text NOT NULL,
  phone       text NOT NULL,
  email       text,
  budget      text,   -- '150만원 미만' | '150~300만원' | '300~500만원' | '500만원 이상' | '미정'
  message     text NOT NULL,
  status      text NOT NULL DEFAULT 'new',  -- 'new' | 'checked' | 'done'
  created_at  timestamptz DEFAULT now()
);

-- RLS: 인증된 사용자만 SELECT/UPDATE/DELETE 가능
-- INSERT는 anon 허용 (문의 폼 제출)
```

## 9. 용어 정의

| 용어 | 의미 |
|------|------|
| Magic Link | 이메일로 전송되는 1회용 로그인 링크. 비밀번호 없이 인증. |
| CTA | Call To Action. "문의하기"처럼 사용자 행동을 유도하는 버튼/링크 |
| RLS | Row Level Security. Supabase DB에서 사용자별 접근 권한을 행 단위로 제어 |
| LCP | Largest Contentful Paint. 페이지 핵심 콘텐츠 로딩 완료 시간 (Core Web Vitals) |
| Framer Motion | React 애니메이션 라이브러리. Apple 스타일 부드러운 전환에 사용. |
